# Use Case: AI-Assisted Requirements Generation

## Problem
Business requirements are often:
- Incomplete
- Ambiguous
- Time-consuming to refine

## Solution
Use AI to transform:
- User stories
- Meeting notes
- Stakeholder inputs

Into structured requirements.

## When to Use
- Early discovery phase
- Backlog refinement
- Documentation uplift

## Benefits
- Faster requirement drafting
- Improved consistency
- Reduced ambiguity

## Risks
- Hallucinated requirements
- Missing domain specifics

## Mitigation
- Human validation required
- Use structured prompts
